## 一、三档定价

| 方案 | 价格 | 查询次数 | 核心卖点 |
|------|------|---------|---------|
| **免费版** Free | $0 | 5次/天 | 体验完整功能，限频 |
| **专业版** Pro | $9.9/月 | 无限 | API接入 + Markdown报告 |
| **企业版** Enterprise | $49/月 | 无限 | 定制规则 + 白标 + 批量 |

### 为什么定 $9.9？

- 心理关口：$10 以下属于"冲动消费"区间
- 对比参照：TokenSniffer $99/月、GoPlus $299/月，我们便宜 10-30 倍
- 目标用户是个人交易者和小团队，$9.9 完全可接受
- 如果将来提价，可以涨到 $19.9/月（功能增加后）

### 企业版 $49 的逻辑

- 白标（在自己网站嵌入我们的扫描结果）是企业的刚需
- 定制检测规则对交易所/钱包来说价值远大于 $49
- 这个价位仍然比竞品便宜 5-10 倍

## 二、收款方式：Gumroad 订阅码

### 为什么不用 Stripe？

- Stripe 不支持中国大陆个人/个体户注册
- 需要境外公司 + 境外银行账户
- 注册流程复杂（EIN、ITIN 等）

### 为什么用 Gumroad？

- 支持 **PayPal 收款**（PayPal 中国大陆账户可用）
- Gumroad 自动处理：税务、发票、退款、地区定价
- 费用：Gumroad 抽 10%，PayPal 抽约 4.4% + $0.30
- 到手：$9.9 → 约 $8.3（合理）

### 操作流程

```
老板在 Gumroad 创建产品（定价 $9.9）
     ↓
系统生成订阅码（PRO-XXXX-XXXX-XXXX）
     ↓
Gumroad 购买完成后，自动/手动发送订阅码给买家
     ↓
买家在网站输入订阅码 → 自动激活专业版
     ↓
买家下个月续费 → Gumroad 再买一次 → 获得新码 → 激活续期
```

### Gumroad 产品设置

1. 产品类型：Digital Product
2. 价格：$9.9（Pro）/ $49（Enterprise）
3. 描述：附上功能对比
4. 购买后内容：发送订阅码 + 使用说明
5. 支持按月订阅（Gumroad 支持 recurring）

## 三、订阅码机制详解

### 生成

```bash
# 管理员在本地或服务器执行
node scripts/generate-codes.js 50 pro 1
# 输出 50 个专业版 1 个月订阅码

node scripts/generate-codes.js 10 enterprise 3
# 输出 10 个企业版 3 个月订阅码
```

### 格式

- 专业版：`PRO-XXXX-XXXX-XXXX`
- 企业版：`ENT-XXXX-XXXX-XXXX`

### 有效期

- 订阅码本身不过期（直到被使用）
- 激活后按购买月数计算到期时间
- 同级别续费：在现有到期时间上叠加
- 不同级别升级：从激活日开始计算

### 防滥用

- 每个订阅码只能使用一次
- 使用后标记为已用（记录使用者和时间）
- 管理员可查询未使用的订阅码数量

## 四、免费版策略

### 为什么保留免费版？

- **引流**：免费用户是付费用户的来源（转化漏斗）
- **SEO**：免费工具容易获得外链和搜索流量
- **口碑**：用户用了觉得好，会推荐给别人
- **数据**：免费用户的使用数据帮助改进产品

### 免费版限制

- 每天 5 次查询（足够认真分析 1-2 个币）
- 不支持 API 接入
- 不带 Markdown 报告导出
- 响应速度略慢（可接受范围）

### 付费转化触发点

当用户遇到以下场景时，自然想付费：
1. 今天额度用完了，但还想再查一个币 → "升级即可无限查询"
2. 想把分析结果接入自己的交易 Bot → "专业版支持 API"
3. 想批量扫 10 个新币 → "企业版支持批量分析"

## 九、FAQ

**Q: 用户买了不续费怎么办？**
A: 免费版仍然可用（5次/天），但 API 调用会中断。用户会有痛感，自然会续费。同时可以发邮件提醒。

**Q: 有人共享 API Key 怎么办？**
A: 速率限制已经在服务端做了（30次/分钟/全局），如果发现异常使用可以封 Key。专业版可以考虑加设备限制。

**Q: 退款的怎么办？**
A: Gumroad 处理退款，我们只需在后台手动停用该订阅码对应的用户权限即可。

**Q: 要不要做年付？**
A: 建议做。$99/年（相当于 8.3/月，省 16%），有利于提前锁定现金流。
